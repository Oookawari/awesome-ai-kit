# 每日科研简报 Automation 创建 Prompt

创建一个每天定时执行的科研情报任务，从论文、技术报告、博客和社区讨论中筛选少量高价值内容，并维护历史推荐记录。

## Prompt

````text
请为我创建一个**每日定时科研简报（Daily Research Brief）**，每天上午固定时间执行。

## 工作目标

每天自动检索并整理我所关注研究领域的最新科研动态，帮助我快速了解值得阅读的新论文、技术报告、博客和社区讨论，而不是简单罗列搜索结果。

你的目标不是搜索尽可能多的内容，而是充当一名**科研情报助手（Research Scout）**，每天筛选出最值得关注的少量内容，帮助我持续跟踪领域发展。

---

## 检索范围

请优先搜索以下来源（不限于）：

### 论文平台

- arXiv
- alphaXiv
- DBLP
- ACL Anthology
- OpenReview
- Hugging Face Papers
- Papers With Code
- Semantic Scholar

### 技术报告

包括但不限于：

- OpenAI
- Anthropic
- Google DeepMind
- Meta AI
- Microsoft Research
- NVIDIA Research
- Apple ML Research
- 阿里、腾讯、字节、百度等公司的技术报告

### 技术博客

包括但不限于：

- Hugging Face Blog
- Lilian Weng Blog
- Sebastian Raschka
- OpenAI Blog
- Anthropic Blog
- Google AI Blog
- DeepMind Blog
- Towards AI
- Towards Data Science
- 其他与你认为相关的高质量博客

### 社区动态

例如：

- Reddit（相关板块）
- Hacker News
- X（Twitter）技术讨论
- GitHub Trending（相关项目）
- 知名研究者个人博客

---

## 检索主题

我的主要研究方向为：

【待补充】

除此之外，也请关注以下相关方向：

- 【待补充】

请理解这些方向之间的关联，必要时可以扩展到相邻研究领域，但仍应保证内容与我的研究兴趣高度相关。

---

## 内容筛选原则

每天不要返回过多内容，建议控制在 **4～7 条以内**。质量远高于数量。

优先推荐：

1. 最新发表的重要论文；
2. 最新技术报告；
3. 有代表性的博客；
4. 值得关注的开源项目；
5. 对经典工作的重新分析（适量）。

如果当天没有特别重要的新内容，可以适当推荐：

- 一篇经典论文；
- 一篇高质量综述；
- 一个优秀博客；

帮助我系统学习。

---

## 去重要求

不要每天重复推荐同一篇论文或博客。请维护历史推荐记录。

原则：

- 已推荐内容，不重复推荐；
- 如果某项工作有重大更新（例如论文正式发表、代码开源、技术报告更新），可以再次推荐，并说明更新内容；
- 如果属于同一系列工作的后续版本，也请说明与之前工作的区别。

---

## 每日输出格式

# 今日科研简报（YYYY-MM-DD）

## 一、本日最值得关注（Top Picks）

推荐 1～3 项当天最重要的内容。

对于每项内容，请包含：

### 标题

### 类型

例如：

- Paper
- Technical Report
- Blog
- Open Source
- Discussion

### 来源

例如：arXiv / OpenReview / Hugging Face Blog 等。

### 链接

提供官方链接。

### 推荐原因

说明：

- 为什么值得我关注？
- 为什么与我的研究方向相关？

### 核心内容（3～5 句话）

概括：

- 研究问题
- 方法
- 创新点
- 主要贡献

### 建议阅读指数

使用 ★★★★★ 评分，并说明：

- 必读
- 推荐阅读
- 可选阅读

---

## 二、其他值得关注

简要列出 2～5 条。每条包含：

- 标题
- 来源
- 一句话介绍
- 官方链接

---

## 三、研究趋势观察（可选）

如果当天出现明显趋势，例如：

- 某方向论文突然增多；
- 某公司连续发布相关工作；
- 某热点开始形成；

请用几句话总结。如果没有，则省略。

---

## 输出要求

请做到：

- 优先官方来源；
- 提供可访问链接；
- 不要简单复制论文摘要；
- 用简洁语言概括工作；
- 强调为什么值得我阅读，而不是介绍所有内容；
- 优先高质量、高影响力内容。

最终目标：每天花 **15 分钟** 阅读这份简报，就能了解该领域最值得关注的最新进展，并持续跟踪我的研究方向。
````

## 使用说明

使用前需要补充：

1. 主要研究方向和相关研究方向；
2. 每天执行的具体时间与时区；
3. 历史推荐记录的保存位置。

为了可靠去重，应让 Automation 能够访问持久化的历史记录，例如持续使用同一个任务线程，或将已推荐条目的标题、官方链接和日期保存在专用文件中。若运行环境无法访问某个平台，应优先使用可访问的官方来源，不应编造搜索结果或链接。
