# 内容维护助手

按照仓库的信息架构将材料转换为可复用条目，同时保护隐私、尊重第三方来源并保持导航完整。

## 工作流程

1. 阅读根目录的 `AGENTS.md`、`README.md` 以及目标分类的 `README.md`。
2. 检查输入是否包含个人信息、密钥、私人对话、敏感项目数据或未公开研究内容。
3. 遇到敏感内容时停止写入，指出需要删除或脱敏的部分；不要擅自把敏感原文写进临时文件。
4. 判断材料是自有内容还是第三方内容。无法确认第三方原始来源时，不收录。
5. 按下方规则确定唯一内容类型和目标位置。
6. 搜索现有标题、目录名和近似用途，避免创建重复条目。
7. 创建或更新条目，保持中文说明简洁，保留必要的英文产品名、命令和代码。
8. 更新目标分类的 `README.md`，删除“尚未添加”占位语并加入条目链接。
9. 更新根 `README.md` 的“按目标查找”入口；只添加有实际内容的导航。
10. 检查所有相对链接、目录命名和变更范围，再向用户报告新增位置与任何来源或隐私限制。

## 分类规则

- **Prompt**：可以在新对话中一次提交并独立完成的指令。放在 `prompts/<name>/README.md`。
- **Workflow**：依赖多轮上下文、多个步骤或多个产物的任务。放在 `workflows/<name>/README.md`。
- **Skill**：包含 `SKILL.md` 和可选规则、资源或脚本的可安装能力。自有内容放在 `skills/local/<name>/`；第三方 Skill 只加入 `skills/external.md`。
- **AI Coding 技巧**：AI Coding 工具的使用或配置说明。放在 `coding-tips/<name>/README.md`。
- **模板**：项目初始化或 Agent 指令骨架。分别放在 `templates/project-init/<name>/` 或 `templates/agent-instructions/<name>/`。

如果难以区分 Prompt 和 Workflow，判断内容是否依赖执行后的回答继续推进。需要继续推进就归入 Workflow，否则归入 Prompt。

## 写入规范

- 直接在内容类型下创建条目，不增加工具或场景分类层。
- 目录名使用小写英文、数字和连字符，并描述条目的工作目标。
- 普通知识条目不使用 YAML front matter。
- 每个条目只保留一个权威副本。用 README 交叉链接，不复制正文。
- 自有内容可以完整保存。第三方内容只写简短说明、适用场景和可靠原始链接，不复制全文。
- 不添加状态标签、案例、个人研究材料或无来源收藏。
- 不创建没有实际内容的预留目录。

## 条目结构

Prompt 条目包含：名称、一句话用途、可复制正文、必要说明、更新记录。

Workflow 条目包含：名称、一句话用途、适用场景、预期产物、操作步骤、完成条件、更新记录。

AI Coding 技巧包含：名称、一句话用途、使用方法、注意事项、更新记录。

外部 Skill 条目使用以下形式：

```markdown
### Skill 名称

简短说明用途和适用场景。

- 原始来源：[项目或作者名称](https://example.com/original)
```

## 修改现有内容

- 保留用户原有意图和可复制性，不为了统一格式改写关键行为。
- 重命名或移动条目时同时修正所有引用。
- 删除条目前确认请求确实授权删除，并从所有索引移除链接。
- 不自动初始化额外工具、CI、网站或元数据系统。
